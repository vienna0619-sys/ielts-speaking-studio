# AGENTS.md

本文件定义本仓库的长期开发规则。任何自动化开发代理和贡献者都必须遵守。

## 产品边界

- 产品是 IELTS 口语练习工具，不是官方考试或官方评分系统。
- 不得使用 IELTS 官方商标、受保护界面或声称拥有未公开题库。
- 题目必须是公开样题结构的示例或原创仿真题，并保持文化中立和未成年人适用。
- 数字考官必须是完全虚构身份，不得模仿或冒充现实人物。

## 考试行为

- 正式模拟考试必须由 `lib/core.mjs` 中的显式状态机驱动。
- 不得用一组松散的 React boolean state 替代考试状态机。
- 模拟考试期间不显示中文提示、转写、参考答案、纠错或评分。
- 考官必须正式、中立、克制；不得夸奖、教学、纠错或代替考生回答。
- Part 2 必须包含 60 秒准备和最多 120 秒讲话；短暂停顿不能自动结束回答。
- Part 2 与 Part 3 必须通过结构化字段保持主题关联。

## 隐私与安全

- API 密钥只能存在于服务端环境变量；禁止 `NEXT_PUBLIC_*` 密钥。
- “保存录音”必须默认关闭。用户主动开启前，不得持久化音频。
- UI 必须清楚说明哪些音频会发送到外部 provider。
- 不得要求上传身份证；身份确认只模拟对话流程。
- Provider 失败时必须显示错误并保留状态；禁止静默失败或伪造云端评分。
- 对无法可靠判断的发音证据标为低置信度。

## 工程规则

- 使用严格 TypeScript；新增浏览器 API 必须有明确类型或最小声明。
- 纯领域逻辑放在 `lib/core.mjs`，并提供 `lib/core.d.mts` 类型声明，便于 Node 原生测试。
- 外部 AI 实现必须遵守 `AiProvider` 接口，浏览器不能直接调用带密钥的第三方 API。
- 题库不得写死在 React 页面 JSX 中。
- 历史和设置 schema 有变化时必须提供向后兼容或迁移。
- 保持 Chrome 为首要测试浏览器，同时不破坏基本响应式和键盘访问。

## 必须运行的检查

任何功能改动完成前必须运行：

```bash
npm run typecheck
npm run lint
npm test
npm run build
```

修改以下逻辑时必须补充或更新测试：

- 题库选择与主题去重
- Part 2 计时
- 状态机与 Part 转场
- Part 2/Part 3 关联
- 评分和辅助指标
- 历史记录保存
- Provider 错误映射
- 麦克风权限异常

## 禁止事项

- 禁止提交 `.env.local`、API 密钥、真实用户录音或个人身份数据。
- 禁止把 Mock 分数伪装成云端 AI 或官方评分。
- 禁止删除免责声明或隐藏低置信度标记。
- 禁止在完整模拟考试中加入教学式提示。
- 禁止未经测试修改状态转场表。
